# CONTRIBUTING.md for ce-pipeline-testbed

## Problem Frame

Contributors need clear guidance on how to set up the project, run tests, and submit PRs. Since this is a test/sandbox repo for the CE pipeline, the contributing guide should set expectations accordingly.

## Requirements

- R1. CONTRIBUTING.md exists at the repository root
- R2. Covers development environment setup (git, GitHub access, Claude Code)
- R3. Covers how to run tests (noting the repo's current test situation)
- R4. Documents PR submission guidelines aligned with the CE pipeline workflow
- R5. Clearly states this is a test/sandbox repository

## Success Criteria

- CONTRIBUTING.md exists at repo root
- Covers setup, testing, and PR process
- Reader understands this is a sandbox repo, not production

## Scope Boundaries

- No code of conduct (out of scope for a sandbox repo)
- No issue templates or PR templates
- No CI/CD documentation beyond what's needed for contributors

## Key Decisions

- Keep it concise since this is a lightweight sandbox repo
- Align with existing README content rather than duplicating it
- Reflect the actual project tooling (Git, GitHub, Claude Code, CE pipeline skills)
