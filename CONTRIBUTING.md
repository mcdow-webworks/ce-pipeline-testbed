# Contributing to ce-pipeline-testbed

> **Note:** This is a test/sandbox repository used to exercise and validate the Claude Windows Worker's CE (Compound Engineering) pipeline. Contributions here help improve the pipeline workflow itself, not a production application.

## Setting Up Your Development Environment

### Prerequisites

- [Git](https://git-scm.com/)
- A GitHub account with access to this repository
- [Claude Code](https://claude.com/claude-code) (for running the CE pipeline locally)

### Setup

1. Fork and clone the repository:

   ```bash
   git clone https://github.com/mcdow-webworks/ce-pipeline-testbed.git
   cd ce-pipeline-testbed
   ```

2. Verify your setup by checking the repo status:

   ```bash
   git status
   ```

No additional dependencies or build steps are required — this is a lightweight sandbox repo.

## Running Tests

This repository currently contains shell scripts and documentation artifacts generated by the CE pipeline. There is no formal test suite at this time.

To manually verify a script works as expected, run it directly:

```bash
bash countdown.sh 5
```

If a formal test framework is added in the future, this section will be updated with specific instructions.

## Submitting a Pull Request

### Branch Naming

Create a feature branch from `master` using one of these prefixes:

- `feature/` — new functionality
- `fix/` — bug fixes
- `docs/` — documentation changes
- `refactor/` — code restructuring
- `test/` — test additions or changes

Example:

```bash
git checkout -b docs/update-contributing
```

### PR Process

1. Make your changes on a feature branch (never commit directly to `master`).
2. Write a clear commit message describing what changed and why.
3. Push your branch and open a pull request against `master`.
4. Describe your changes in the PR body — what you changed, why, and how to verify it.

### Using the CE Pipeline

This repo is designed to work with Claude Code's CE pipeline skills. You can trigger the full workflow by:

1. Creating a GitHub issue describing the work to be done.
2. The automated worker picks up the issue, brainstorms, plans, and implements changes.
3. A pull request is opened automatically for review.

You can also run the pipeline locally in Claude Code using `/ce:brainstorm`, `/ce:plan`, and `/ce:work`.

## Questions?

Open an issue in this repository if you have questions about contributing or the CE pipeline workflow.
